# Changelog

所有版本变更记录。格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，版本号遵循 [Semantic Versioning](https://semver.org/)。

发布新版本步骤：
1. 在本文件顶部添加新版本段落
2. 提交代码：`git commit`
3. 打 tag：`git tag v1.x.x`
4. 推送 tag：`git push origin v1.x.x`
5. GitHub Actions 自动构建并发布 Release

---

## [v1.1.1] - 2026-06-16

### 新增
- 检查更新下载显示进度条，安装前引导开启「安装未知应用」权限
- 封面图片本地缓存（内存 + 磁盘），不再每次实时加载
- 搜索时部分源失败给出提示，便于诊断

### 修复
- 详情页进入后默认集 / 历史集自动开始播放
- 详情页视频下方的选集 / 线路 / 源信息区不再消失
- 秉奇影视详情页解析标题与封面（原为空），搜索解析更鲁棒（不依赖站点 class 名 / 属性顺序）
- 搜索结果切换底部标签后保留，不再销毁
- modu 封面在搜索页加载失败的问题（图片缓存后稳定显示）

## [v1.1.0] - 2026-06-15

### 新增
- 源评分模型：按真实播放结果（可用性 / 稳定性 / 实际分辨率 / 加载速度）为每个源打分，失败冷却降权
- 搜索结果按源评分分级展示：优先源 / 可用源 / 其他源
- 多源聚合详情：同一内容合并多个源，详情页内切换源不离开页面
- 详情页播放状态机：进入即自动播放，自动切线路 / 切源，从历史进入恢复上次观看现场
- 本地观看历史与收藏（无需账号）：历史 / 收藏页，进度记录，一键清除
- 播放器按真实分辨率回填评分，显示清晰度角标与网络缓冲状态
- 检查更新：启动自动 + 设置页手动检查 GitHub 最新版本，Android 应用内下载安装
- 设置页 GitHub 仓库 / 问题反馈 / 版权处理点击跳转网页
- 全新深色主题 UI（搜索 / 历史 / 收藏 / 详情 / 设置五页），统一 KMP Compose

### 修复
- 详情页进入后默认集 / 历史集自动开始播放，无需手动点一次
- 详情页选集 / 线路 / 源信息区不再消失
- 收藏状态判断口径统一，修复重进详情收藏图标恒显示未收藏
- 播放中切换线路 / 源不再被静默丢弃

### 变更
- APK 命名规范为 `veya-{版本}-{buildType}.apk`
- debug 与 release 统一签名，便于覆盖安装
- CI 改为发布 release 签名包

## [v1.0.0] - 2026-06-12

### 新增
- 搜索页：关键词搜索，支持多数据源并发查询
- 搜索历史：自动记录最近 20 条，点输入框展示，支持单条删除
- 详情页：多线路 / 选集切换，当前集高亮
- 内嵌播放器：ExoPlayer HLS 播放，无广告无跳转
- 手势控制：单击切换控制栏 / 水平滑动快进快退 / 长按 2 秒 2× 倍速
- 全屏模式：横屏 + 隐藏状态栏导航栏，切换不中断播放
- 数据源：模板影视（caiji.moduapi.cc）苹果CMS v10 JSON API
